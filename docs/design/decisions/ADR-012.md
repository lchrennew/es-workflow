# ADR-012：引入 MetadataBase 作为元数据基类

## 背景与问题
Workflow 与 WorkflowStateEmitter（以及 Prefetcher）都采用 `kind/name/metadata/spec` 的声明式配置结构，且 metadata 字段高度一致（title/tags/审计字段/version）。如果分别定义，会产生重复与不一致风险。

## 决策
引入 `MetadataBase` 作为元数据基类：
- `WorkflowMetadata` 继承 `MetadataBase`
- `WorkflowStateEmitterMetadata` / `PrefetcherMetadata` 继承 `MetadataBase`

## 决策原因
1. **消除重复**：统一通用元数据字段定义，减少重复维护。
2. **一致性**：跨配置资源的元数据含义与约束保持一致。
3. **便于扩展**：未来新增通用元数据字段时（如 tenant/namespace），只需在基类扩展一次。
