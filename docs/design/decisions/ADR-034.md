# ADR-034：移除 WorkflowTask.updatedAt（无需记录更新时间）

## 状态
已采纳（2026-05-05）

## 背景
WorkflowTask 最初设计包含 `updatedAt` 字段（UTC 字符串），用于记录任务的最后更新时间。

Task 的更新场景包括：
- 状态变更（Initialized → InProgress → Completed/Ignored）
- 添加 request
- request 收到 response
- 参数更新（livingParameters、outputParameters）

## 决策
移除 `WorkflowTask.updatedAt` 字段。

## 决策原因
**核心原因：没有必要记录这个信息**

1. **无业务价值**：Task 的更新时间对工作流引擎的运行逻辑没有实际作用
2. **无审计需求**：如需审计 task 的变更历史，应通过 WorkflowEvent 实现，而不是依赖单一的 updatedAt 时间戳
3. **无查询需求**：实际业务中不需要"查询最近更新的 task"这样的场景
4. **减少维护成本**：每次更新都要维护 updatedAt，增加实现复杂度且容易遗漏

**注意**：这个决策与 ADR-028/032/033 的"从 ObjectId 推导时间"原则不同。这里不是"可推导所以不需要"，而是"根本不需要这个信息"。

## 并发控制
如需对 task 进行并发控制（防止并发更新相互覆盖），应采用：
- 内部版本号（version 字段）
- 数据库层面的乐观锁机制（如 MongoDB 的 `findAndModify`）

而不是依赖 updatedAt 字段。

## 影响
- WorkflowTask 模型更简洁
- 减少字段维护成本
- 并发控制需要通过其他机制实现（版本号或数据库层面）
