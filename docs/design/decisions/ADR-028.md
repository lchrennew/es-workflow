# ADR-028：运行期对象使用 ObjectId 时不再保存 createdAt/occurredAt（时间从 ObjectId 推导）

## 背景与问题
运行期对象（WorkflowTask/WorkflowRequest/WorkflowEvent 等）需要用于审计时间线与排障。通常会希望记录“创建/发生时间”。

在当前模型中，上述对象的 `id` 采用 MongoDB ObjectId（或等价的“带时间戳的 ID”），其高位已编码了创建时间。

因此存在两种方案：
1) 额外字段：`createdAt/occurredAt`（UTC 字符串）  
2) 不加字段：从 `id(ObjectId)` 推导时间

## 决策
只要对象的主标识为 `ObjectId`，则：
- 不再保存 `createdAt`（创建时间）
- 不再保存 `occurredAt`（发生时间）
统一从 `id(ObjectId)` 推导时间。

## 决策原因
1. **避免冗余与不一致**：双写时间字段容易出现不一致（时区/格式/更新顺序）。
2. **简化模型**：减少字段与存储成本，保持事件记录最小化。
3. **实现可行**：ObjectId 可直接反解创建时间，满足“时间线排序/展示”的需求。

## 影响与后果
1. UI 展示需要在查询/渲染层把 `ObjectId.timestamp` 转为可读时间。
2. 仍可保留 `updatedAt`（若对象存在可变更字段且需要显示最后更新时间），因为它无法从 id 推导。
3. 若未来改用不含时间戳的 id 方案（例如 UUIDv4），需要重新评估是否补充 `createdAt/occurredAt`。
