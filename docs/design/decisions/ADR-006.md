# ADR-006：state.conditions（脚本）不通过时触发事件，并将 Task 置为 Ignored

## 背景与问题
对于已被激活的状态，如果其 `state.conditions` 判定不通过：
- 既不希望阻塞流程等待条件变真（部分场景需要自动绕行/跳过）
- 又希望保留“该状态曾被激活并尝试创建任务”的审计痕迹

## 决策
1. 只要匹配迁移 `event`，就激活下游状态并创建 Task（不再对 target 设置条件）。
2. Task 创建后立刻执行 `state.conditions`：
   - 满足：Task 进入 `in-progress`
   - 不满足：将 Task 置为 `Ignored`，并统一触发内部事件 `ignored`

## 决策原因
1. **避免不必要的阻塞**：用事件驱动“自动绕行/自动处理”而不是等待。
2. **语义清晰**：迁移只负责“激活哪些状态”，`state.conditions` 负责“任务是否进入处理态”。
3. **保留可观测性**：Ignored 作为完成态的一种，保留了任务创建与被跳过的记录，便于审计与问题排查。

## 强校验规则（已确认）
1. 若 `WorkflowState.conditions` 非空，则必须存在一条 `transition.event="ignored"` 的迁移，用于定义条件不通过时的后续推进路径。
2. `ignored` 事件对应的迁移必须在流程编排上收敛到后续 join（汇合）状态，保证并行分支可收敛。
