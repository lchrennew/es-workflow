# ADR-030：移除 WorkflowRequest.status（以 voidInfo + responses 推导状态）

## 背景与问题
WorkflowRequest 需要表达“是否仍有效、是否已决策、是否已作废”等状态，用于：
- 规则脚本统计（仅统计决策类响应、忽略作废请求）
- 引擎语义校验（作废请求不再接收响应）
- UI/审计展示

在当前模型中：
- `WorkflowRequest.responses` 为日志，可从中判断是否已出现 `kind="decision"` 的响应
- `WorkflowRequest.voidInfo` 记录作废信息，可表达“作废（留痕）”事实

因此 `WorkflowRequest.status` 属于可推导字段。

## 决策
移除 `WorkflowRequest.status` 及其枚举 `WorkflowRequestStatus`。

以如下原始字段推导请求状态：
- **是否作废（voided）**：`voidInfo != null`
- **是否已决策（decided）**：`responses` 中存在 `kind="decision"` 的响应（且最多一个、必须最后）
- **是否仍可响应（open）**：`voidInfo == null` 且尚无决策类响应

## 决策原因
1. **避免冗余与不一致**：状态可由原始事实推导，双写易产生不一致。
2. **职责边界清晰**：request 查询/筛选不属于工作流引擎的领域职责，无需为查询性能引入冗余状态字段。
3. **遵循奥卡姆剃刀**：能推导则不存储。

## 影响与后果
1. 规则脚本应以 `voidInfo` 判断作废、以 `responses.kind` 判断决策响应。
2. 若未来确需对 requests 做高频筛选/分页（引擎职责发生变化），再评估引入冗余字段或二级索引。
