# ADR-002：多目标迁移的条件表达（toStates -> targets + when）（已废弃）

> 状态：已废弃（历史记录保留用于追溯）  
> 替代方案：见 ADR-013（移除 target.conditions/when，仅保留 state.conditions（脚本）做任务门控），以及当前 `targets + prefetchers` 的结构定义。

## 背景与问题
同一事件可能激活多个后续状态，并且部分目标状态需要满足前提条件（例如“打回修改后再次提交，需要跳过某些 states”）。

## 决策
将迁移目标从 `toStates: List<String>` 调整为：
- `targets: List<WorkflowTransitionTarget>`
  - `state: String`（目标状态标识，引用 `WorkflowState.name`）
  - `when: String?`（可选条件表达式，后续已被 ADR-003 替代）

> 备注：由于后续选择“仅保留 State.conditions（脚本）”，`WorkflowTransitionTarget` 不再承载条件字段，本 ADR 仅作为历史记录保留（见 ADR-013）。

## 决策原因
1. **目标级条件**：条件往往针对“某个目标状态是否激活”，而不是针对整个事件；放在 target 上粒度更合适。
2. **可表达跳过**：通过不同 target 的 `when`，可以在同一事件下直接激活更后序状态，从而实现“跳过中间状态”。
3. **便于扩展**：未来若需要为目标增加更多信息（如激活模式、优先级、产出变量映射等），可在 `WorkflowTransitionTarget` 上扩展而不破坏结构。

## 未决项（后续你来定）
- 条件表达式语言（例如 CEL/SpEL/JEXL/自定义）与可访问上下文（运行变量、历史状态、用户等）。
- 多个 target 同时满足条件时的语义：全部激活（并行）/按优先级/互斥选择等。
