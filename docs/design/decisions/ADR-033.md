# ADR-033：移除 WorkflowResponse.time（从 ObjectId 推导时间）

## 状态
已采纳（2026-05-05）

## 背景
WorkflowResponse 最初设计包含 `time` 字段（UTC 字符串），用于记录响应时间。

但 WorkflowResponse.id 使用 ObjectId，可以直接从中提取时间戳。根据 ADR-028 的原则（使用 ObjectId 的实体不需要单独的 createdAt 字段），response.time 字段是冗余的。

## 决策
移除 `WorkflowResponse.time` 字段。

响应时间从 `response.id`（ObjectId）提取即可。

## 决策原因
1. **与 ADR-028 原则一致**：使用 ObjectId 的实体不需要单独的时间字段
2. **避免冗余**：遵循奥卡姆剃刀原则，如果原始数据已经可以判定，就无须冗余存储
3. **避免数据不一致**：减少维护成本，避免 time 与 id 中的时间戳不一致的风险
4. **简化模型**：减少字段数量

## 业务时间的处理
如果外部系统需要记录"业务响应时间"（与系统接收时间不同），应该通过 `response.payload` 中的业务字段来表达，而不是用系统级的 time 字段。

例如：
```json
{
  "action": "APPROVE",
  "kind": "decision",
  "payload": {
    "comment": "同意",
    "businessTime": "2026-05-05 14:30:00"  // 业务时间作为 payload 的一部分
  }
}
```

## 影响
- WorkflowResponse 模型更简洁
- 响应时间统一从 response.id（ObjectId）提取
- 如需记录业务时间，应放在 payload 中作为业务数据
