# ADR-032：移除 WorkflowRun.createdAt/updatedAt 和 runNumber，id 改为 `<工作流标识>.<ObjectId>`

## 状态
已采纳（2026-05-05）

## 背景
WorkflowRun 最初设计（**现已废弃**）包含：
- `id`：自定义格式 `<workflowNameWithDots>.<runNumber>`（如 `order.approval.3`）
- `runNumber`：该工作流下的自增序号（从 1 开始）
- `createdAt` 和 `updatedAt`：时间字段

这种设计存在以下问题：
1. **自增机制复杂**：需要维护每个工作流的独立计数器，增加实现复杂度
2. **时间字段冗余**：虽然自定义 id 无法提取时间戳，但可以从关联对象推导
3. **runNumber 业务价值低**：运行编号对实际业务意义不大

## 决策
1. **WorkflowRun.id 改为 `<工作流标识>.<ObjectId>` 格式**：
   - 工作流标识来自 `Workflow.name`，并将 `/` 替换为 `.`
   - ObjectId 部分用于唯一标识该次运行
   - 示例：`order.approval.507f1f77bcf86cd799439011`
   - 可直接从 ObjectId 部分提取创建时间戳
   
2. **移除 runNumber 字段**：
   - 不再需要工作流级别的运行编号
   - 如需按工作流查询运行历史，通过 `name` 字段过滤即可

3. **移除 createdAt 和 updatedAt 字段**：
   - **创建时间**：直接从 `run.id` 的 ObjectId 部分提取时间戳
   - **更新时间**：从 `run.events` 或 `run.tasks` 的最后一条记录的 ObjectId 提取时间戳

## 影响
- WorkflowRun 模型更简洁，减少字段维护成本
- 无需维护自增计数器，降低实现复杂度
- id 仍包含工作流名称信息，便于识别和调试
- 查询更新时间需要额外的推导逻辑（从最后一条 event/task 的 ObjectId 提取）
- 与 ADR-028（运行期对象使用 ObjectId 时不保存 createdAt）的原则完全一致
