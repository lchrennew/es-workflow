# ADR-017：TMP_ 前缀参数为临时参数（不输出、不后传）

## 背景与问题
任务执行过程中会产生大量仅用于中间计算/临时判断的参数。如果这些参数跟随 task 输出并合并到 run.livingParameters：
- 会污染全局上下文，增加后续条件/脚本的噪声
- 容易引入意外依赖（后续任务错误地使用临时变量）

## 决策
- 参数 key 以 `TMP_` 开头的，视为临时参数：
  - Task 结束时 **不输出**（不进入 `task.outputParameters`）
  - Task 结束时 **不后传**（不合并到 `run.livingParameters`）

## 决策原因
1. **隔离中间过程**：明确临时变量生命周期，仅在任务运行期存在。
2. **降低耦合与噪声**：减少 run.livingParameters 被临时字段污染，提升可读性与可治理性。
3. **减少误用风险**：通过命名约定避免后续状态/脚本依赖短生命周期变量。
