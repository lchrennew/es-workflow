# ADR-009：结束状态采用保留名 "end"

## 背景与问题
WorkflowRun 需要一个明确、可判定的“结束条件”，否则运行循环无法统一收敛（尤其在存在有环图时）。

## 决策
- 约定每个工作流配置中必须存在且仅存在一个系统保留状态：`WorkflowState.name = "end"`。
- 该状态用户不可删除、不可修改。
- 当运行到达（激活）该状态时，WorkflowRun 进入 Completed（结束）。

## 决策原因
1. **结束判定统一**：避免依赖“无可运行任务/无激活状态”等间接条件导致歧义。
2. **适配有环图**：即使存在循环，只要能到达 end，就能明确结束。
3. **实现简化**：引擎只需识别保留状态即可收敛运行。

## 影响与后果
- 需要保证 `"end"` 不被用户用于业务含义的自定义状态名（保留字）。
