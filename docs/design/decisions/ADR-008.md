# ADR-008：初始状态采用保留名 "initial"

## 背景与问题
WorkflowRun 启动时需要一个明确的“初始激活状态”，以便形成确定的启动行为（激活状态、创建任务、进入运行循环）。

## 决策
- 约定每个工作流配置中必须存在且仅存在一个系统保留状态：`WorkflowState.name = "initial"`。
- 该状态用户不可删除、不可修改。

## 决策原因
1. **降低建模复杂度**：无需在 spec 中额外引入 `initialStateName` 字段或 `isInitial` 标记。
2. **启动行为确定**：引擎可直接定位初始状态，避免“无初始态/多初始态”导致的不确定性。
3. **便于一致校验**：可在配置校验阶段强制检查保留状态是否存在。

## 影响与后果
- 需要保证 `"initial"` 不被用户用于业务含义的自定义状态名（保留字）。
